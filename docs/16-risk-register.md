# Risk Register

| Risk | Severity | Current control | Next action |
| --- | --- | --- | --- |
| Domain breadth may hide inconsistent invariants | High | Real PostgreSQL Source-to-OutcomeFeedback chain plus module integration tests | Add mutation/property tests and repository contracts per aggregate |
| Tenant scope omission | Critical | PostgreSQL RLS and non-owner transactions cover A-H facts; Phase H E2E verifies cross-tenant intelligence and feedback reads | Add automated policy inventory checks for every future migration |
| Money posting error | Critical | Integer minor units, account locks, deferred debit/credit validation, immutable runtime history, snapshots, idempotency, and PostgreSQL integration tests | Add external payment reconciliation and multi-currency FX policy before production money movement |
| Workflow side effects repeat | High | Persisted step leases, attempts, retry schedules, signed result receipts, Inbox deduplication, and lease ownership checks | Add multi-node dependency release, compensation, and crash-recovery concurrency tests |
| SSRF through crawler redirects/DNS | Critical | Documented deny policy | Implement resolver pinning and redirect tests in crawler worker |
| Portal skeletons may outpace backend authorization | Medium | All seven portals build; five operational portals use authenticated APIs and command permissions | Add customer and developer documentation workflows only when backed by production contracts |
| Session theft or stale privileges | High | HttpOnly SameSite cookies, hashed tokens, expiry, revocation, membership resolution, and production secure-Cookie release check | Add session rotation and membership-change revocation |
| Outbox delivery outage | High | Leases, backoff, dead-letter alerts, health summary, audited admin replay, and production release check | Route critical alerts to an on-call channel and add SLA ownership |
| Redis and MinIO images were not pulled | Medium | Compose contracts are present; PostgreSQL is healthy | Retry image pulls when Docker Hub TLS is stable, then add health integration tests |
| Shared portal navigation uses placeholder anchors | Medium | Admin/operator H workspaces and reseller/supplier/marketplace operational workspaces are connected | Complete authenticated customer and developer documentation workflows |
| Published version references may drift | Critical | Quote creation requires an active Publication; OrderItem copies immutable product, SKU, workflow, pricing, and routing identifiers; SupplierContract is persistent and versioned | Bind the selected supplier contract version into route decisions and immutable OrderItem snapshots |
| Operational charge may be mistaken for posted revenue | Critical | Usage, ProviderCost, CustomerCharge, and LedgerEntry are separate; explicit posting atomically advances CustomerCharge and writes balanced entries | Require finance views to display calculated and posted amounts separately in every future portal |
| Administrative wallet adjustment may be abused | Critical | Dedicated `finance.adjust` permission is admin-only; every adjustment is idempotent, balanced, audited, and emitted through Outbox | Add dual approval and policy limits before production funding adjustments |
| Reconciliation discrepancy may remain unresolved | High | Runs persist per-fact expected/actual values and discrepancy records without changing ledger history | Add assignment, SLA, evidence, and adjustment-approval workflow |
| Manual execution transition may overstate completion | High | Browser result states/output are blocked; Provider-bound HMAC Adapter receipts require active workflow lease ownership | Add Adapter key rotation, mTLS, artifact signatures, and provider reconciliation evidence |
| Suppressed contact may be planned or quota-counted | Critical | Tenant-keyed active suppression is checked before quota reservation; suppressed attempts persist as `blocked`; cancellation releases reservations | Add provider-level bounce/complaint ingestion through signed, replay-protected webhooks |
| Campaign operator may self-approve outbound scope | Critical | Separate `campaign.write` and `campaign.approve` permissions; approval binds the exact campaign version | Add dual approval and policy thresholds before enabling any delivery adapter |
| External content may trigger delivery or privileged actions | Critical | Conversation/Proof remain untrusted JSON; execution results have trusted Adapter ingress; outreach delivery stays disabled | Design a separate approved, suppression-aware signed delivery ingress before real sending |
| Deal, Quote, and Order facts may diverge | High | `quotes.growth_deal_id` references canonical Deal; server derives customer from Deal; proposal/win gates check Quote state; Order remains transaction-owned | Add quote-version amendment and multi-deal merge/split policies before advanced CRM workflows |
| Channel ownership may be captured or self-approved | Critical | One active protected owner per Lead/Customer, independent transfer permission, requester/reviewer actor check, audit, Outbox, and integration tests | Add expiry worker, conflict arbitration SLA, and reseller-scoped membership filtering |
| Supplier portal may imply a second payable ledger | Critical | Supplier views join canonical ProviderPayable and Settlement through Provider ownership; no supplier payment table or payout command exists | Add supplier invoice evidence and reconciliation without changing finance ownership |
| Marketplace artifact may escape review | Critical | Immutable ListingVersion, separate reviews, sandbox record, quality gate, and takedown-only removal | Bind a dedicated sandbox worker identity and artifact signature verification before external listings |
| Marketplace payout may move money prematurely | Critical | `marketplace.payout` defaults disabled; payout reserve has no public command and no external money adapter | Design dual-approved payout posting against the canonical ledger before enabling the feature |
